Brief Description

Concur is a purely functional language with a philosophy of simplicity, concurrency, and ease-of-use. Concur allows programmers to easily take advantage of parallelism in purely functional programs through support for implicit lazy-future evaluation. In addition, Concur features an effect system that allows reasoning about concurrency, state and effects in an intuitive and selectively deterministic manner through a variant of Functional Reactive Programming [FRP] termed Effect Driven Functional Reactive Programming [ED-FRP]. Concur is strongly, but optionally typed, featuring type inference, subtyping, parametric types, ad-hoc polymorphism and parametric polymorphism. The effect is that Concur can provide static guarantees where demanded, but also explicitly allow dynamically-typed programs.

Concur combines simplicity with power, but Concur is not designed for the type theorist or the category theorist. Concur is designed for the programmer.
